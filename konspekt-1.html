Что есть в HTTP запросе и ответе?

HTTP-запрос: Квест ждет

  1. Host - url сайта (example.com)
  2. Accept - ожидаемые от сервера данные
  3. Accept-encoding - шифровка/архивирование ожидаемых данных
  4. Accept-language - язык ожидаемых данных
  5. Connection - статус, определяющий, останется ли сетевое соединение открытым
  6. User-Agent - Клиент пользователя

HTTP-ответ:

  1. Версия протокола HTTP, статус страницы
  2. Content-type - содержание "конверта"
  3. Connection - статус, определяющий, останется ли сетевое соединение открытым
  4. Content-encoding - Архивирование/шифрование отправляемых данных
  5. Date - полная дата(удивительно, да?)
  6. Server - тут объяснять не стоит
  7. Transfer-encoding - метод шифровки и передачи данных

Термины

В технических курсах без терминов и определений не обойтись. Вот несколько из них:

Cookie:
Небольшой фрагмент данных, отправленный веб-сервером и хранимый на компьютере пользователя.
HTTP:
Протокол прикладного уровня передачи данных изначально — в виде гипертекстовых документов в формате «HTML», в настоящий момент используется для передачи произвольных данных.
HTTPS:
Более защищённый HTTP. Да, это все, а что?
CSS:
Язык таблиц стилей, который позволяет прикреплять стиль (например, шрифты и цвет) к структурированным документам .
